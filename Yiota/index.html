Next Generation Asset Investment Platform

Overview

Our project is about creating an Investment Platform that will be used by Researchers as well as Analysts of Hedge Funds. The main task will be to make a system that will be used in an investment firm to make investment predictions through complex analysis using model generation and machine learning. We will be creating a system that will be able to model tradable products (e.g. commodities) by configuring certain variables and algorithms. We will be collecting data of various formats, cleaning them and producing a unified format. This data will be fed into a system of neural networks to make a prediction in the future change of stock prices.

Research

Scraping

To scrape data we had to make a choice between two popular python based scraping tools, BeautifulSoup and Scrapy. BeautifulSoup is a content extractor i.e. a library for parsing and extracting data from HTML web pages. It needs to be coupled with an external package such as urlib2 or requests to download the web page and then can parse the HTML source code. It is great for creating a one-off script and is relatively easier to learn and implement. Scrapy, on the other hand, is a more complete scraping platform and can perform multiple tasks such as downloading, cleaning and saving data. It is asynchronous in nature which means it does not perform tasks in a sequence but concurrently. Scrapy is used to create bots called spiders which can then crawl through web pages as per the developer’s definition and requirements. Scrapy, however harder to master, is an end-to-end web scraping framework and is more extensible than BeautifulSoup and as BeautifulSoup does not support multiprocessing without importing, Scrapy is much more efficient as well. Hence, we decided to work with Scrapy.


Database

Two options for the database were MongoDB and MySQL. MySQL is a full-featured open-source relational database management system (RDBMS) owned by Oracle Corporation. It stores data in tables that are grouped into a database and uses Structured Query Language(SQL) for defining and manipulating data. SQL can be restrictive as it follows predefined schemas to determine the structure of your data. MongoDB is a NoSQL database and the documents are created a Binary JSON format. It was released in 2009 by 10gen. MongoDB can store related data together which improves speed. MongoDB uses the MongoDB query language and is schema-free, which allows the user to create documents without having to define the structure of the document first. These documents can also be easily changed by adding or deleting fields. Both the database systems are great options but MongoDB is known to work well with real-time analytics and we require a dynamic schema, so we ended up selecting MongoDB as our database.


Web App

To create the web application, we had to choose amongst three frameworks, Angular, React and Node. AngularJS was launched in 2009 by google and is a client-side web framework used to develop single page web applications. AngularJS uses HTML to define a web app’s user interface, this simplifies the web development process. It is also great for handling dependencies and testing. ReactJS is more like a library rather than a framework and can be used to create amazing user interfaces. React differs from other libraries and frameworks by using a Virtual Document Object Model. Virtual DOM can be considered React’s local and simplified copy of the HTML DOM. It allows React to do its computations within this abstract world and skip the “real” DOM operations, often slow and browser-specific. It is very easy to create UI test cases and the components in react can be easily reused. Node.JS is a very different framework from the other two and can be considered as more of a server itself. It is a Run time environment based on Google Chrome V8 Javascript Engine and enables Javascript to run on the computer itself rather than using a browser. As we are using MongoDB which creates JSON format files, it can be easily integrated with NodeJS. Also, because we did not require a very fancy User Interface, React and Angular were not essential. However, we did require the flexibility that Node gives for both client side and server side applications, therefore we decided to use Node.JS.
